Files other than this README are generated by `../grep-algorithms.sh`.

Do not commit changes here without also updating ALGORIHMS.md. This helps keep
the code and the documentation in sync.

Until we have better means to extract lists of algorithms and properties,
keeping a list manually in `../../ALGORITHMS.md` is a practical solution. To
assist that, a crude grep of the code (done in `../grep-algorithms.md`)
provides a snapshot of what has been considered for the ALGORITHMS file. This
mechanism should catch most cases of added implementations and algorithms, and
not be too burdensome by requiring constant acknowledgement that performed
changes did *not* affect that lst.
